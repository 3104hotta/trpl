# rust-learning

『[The Rust Programming Language 日本語版](https://doc.rust-jp.rs/book-ja/)』を毎日少しずつ読み進めるための学習用リポジトリです。
各章ごとに独立した Cargo プロジェクトを作成し、写経・実験コードを蓄積していきます。

---

## 📂 リポジトリ構成ルール

1 つのルートリポジトリの直下に、章ごとの Cargo プロジェクトを並べる **パターンA** で運用します。

```
rust-learning/
├── .gitignore
├── README.md
├── ch01-getting-started/
├── ch02-guessing-game/                  # プロジェクト系（通常構成）
└── ch03-common-programming-concepts/    # 概念解説系（src/bin/ 構成）
    ├── Cargo.toml
    └── src/
        ├── main.rs
        └── bin/
            ├── 01_variables.rs
            └── 02_data_types.rs
```

### 章タイプの使い分けガイドライン

| タイプ | 対象章（例） | 管理方法 |
| :--- | :--- | :--- |
| **プロジェクト系**（成果物を作る章） | 第2章, 第12章, 第20章 | 通常の Cargo プロジェクトとして `src/main.rs` / `src/lib.rs` を書籍の指示通りに作り込む |
| **概念解説系**（短い実験コードが続く章） | 第3〜6章, 第8〜10章, 第13章, 第15〜16章, 第18章 | 章ごとに 1 つの Cargo プロジェクトを作り、節ごとのサンプルは `src/bin/NN_topic.rs` として追加 |

### 章プロジェクトの作成コマンド

```bash
# プロジェクト系の章を新規作成
cargo new ch02-guessing-game

# 概念解説系の章を新規作成（中身は src/bin/ で運用）
cargo new ch03-common-programming-concepts
mkdir ch03-common-programming-concepts/src/bin
```

### `src/bin/` の実行方法

概念解説系の章では、節ごとのファイルを `src/bin/` 配下に置き、ファイル名で個別実行します。

```bash
# ch03 の "変数と可変性" の節を実行
cd ch03-common-programming-concepts
cargo run --bin 01_variables

# "データ型" の節を実行
cargo run --bin 02_data_types

# 該当章のすべてのバイナリ／メイン／テストをビルドだけ確認
cargo build
```

> 💡 `src/bin/FILE.rs` を置くと、Cargo は自動的に `FILE` という名前のバイナリターゲットとして認識します。`Cargo.toml` への追記は不要です。
> ファイル名は `NN_topic.rs`（例: `01_variables.rs`）のように 2 桁ゼロパディング＋スネークケースで統一すると並び順が安定します。
